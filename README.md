# Pulpas-Tropikae
Contexto:
Página web diseñada para una empresa dedicada a la comercialización de pulpas de fruta, enfocada en brindar una experiencia digital práctica, atractiva y fácil de navegar para los usuarios. La plataforma incluye enlaces directos a redes sociales para fortalecer la interacción y comunicación con los clientes, además de un carrito de compras que facilita el proceso de selección y adquisición de productos en línea. También cuenta con un catálogo organizado de pulpas de diferentes sabores y presentaciones, acompañado de un espacio exclusivo con recetas variadas, donde los usuarios pueden encontrar ideas creativas y saludables para preparar bebidas, postres y otras preparaciones utilizando las pulpas de fruta.
Rol de QA:
En el desarrollo y validación de la página web enfocada en la comercialización de pulpas de fruta, el rol del QA (Quality Assurance) es fundamental para garantizar que la plataforma funcione correctamente, sea intuitiva para el usuario y cumpla con los estándares de calidad establecidos. Entre las actividades principales del QA se encuentraron la ejecución de pruebas manuales y automatizadas sobre funcionalidades como el carrito de compras, el catálogo de productos, los enlaces a redes sociales y el módulo de recetas. Estas pruebas permiten identificar errores funcionales, fallos de navegación, problemas de compatibilidad y comportamientos inesperados antes de que el sistema sea publicado.
Dentro de las pruebas manuales, se debe validar procesos críticos como el registro e inicio de sesión de usuarios, la búsqueda y filtrado de productos, la correcta visualización de imágenes y descripciones, el flujo de compra, la actualización del carrito y el acceso a enlaces externos. También debe realizar pruebas de usabilidad, pruebas responsive en diferentes dispositivos y navegadores, así como pruebas exploratorias para detectar posibles inconsistencias en la experiencia del usuario.
En las pruebas automatizadas, implemento scripts para validar procesos repetitivos y optimizar tiempos de ejecución, especialmente en funcionalidades críticas del sistema. Estas pruebas ayudan a garantizar estabilidad en futuras actualizaciones de la plataforma.
Para la gestión y seguimiento de incidencias se utiliza Jira, herramienta que permite documentar errores, asignar tareas, realizar seguimiento del estado de los tickets y organizar el trabajo bajo metodologías ágiles como Scrum. Asimismo, el QA puede emplear Postman para realizar pruebas de APIs, validando el correcto intercambio de datos entre el frontend y backend, comprobando respuestas del servidor, métodos HTTP, autenticación y tiempos de respuesta.
Adicionalmente, pueden utilizarse herramientas complementarias como Selenium para automatización de pruebas web, Google Chrome DevTools para depuración y análisis del comportamiento del sitio, y TestRail para la documentación y organización de casos de prueba. Todo este proceso permite asegurar que la página web ofrezca una experiencia confiable, segura y eficiente para los usuarios finales.
